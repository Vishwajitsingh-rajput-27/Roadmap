HTML ‚Äî The Structure

HTML is like the skeleton of a webpage. It tells the browser what to show. Think of it as building blocks: <h1> is a heading, <p> is a paragraph, <div> is a container box.

CSS ‚Äî The Styling
‚ñº

CSS is like the paint and decoration. It controls how things look ‚Äî colors, fonts, spacing, layout. The biggest things to master are: Selectors (targeting elements), Box Model (margin, padding, border), and Flexbox (the #1 way to lay things out ‚Äî learn this well).

JavaScript ‚Äî The Behavior
‚ñº

JavaScript makes your page interactive. This is the meat of frontend development. How the Web Actually Works
‚ñº

Before writing backend code, understand the big picture. When you type a URL, your browser sends a Request to a Server, which processes it and sends back a Response. This is called the Client-Server Model. Learn about HTTP methods: GET (fetch data), POST (send data), PUT (update), DELETE (remove). Also learn what APIs are ‚Äî they're basically the interface between your frontend and backend.

Build a Server with Node.js + Express
‚ñº

Express.js is a tiny, powerful library that turns Node.js into a web server. Install it with npm install express. Then you can create routes ‚Äî think of them as pages your server can respond to. For example, app.get('/api/users') would return a list of users when someone visits that URL.

Mini Project: Create a simple Express server with 3 routes: one that returns a welcome message, one that returns a list of products (fake data in an array), and one that accepts a POST request to "create" a product. Test it using Postman (a free tool for testing APIs).

Authentication ‚Äî User Login/Signup
‚ñº

This is how your website knows who is logged in. The standard flow is: user submits email + password ‚Üí server checks credentials ‚Üí server creates a JWT token (a secure code) ‚Üí frontend stores it ‚Üí on every future request, the frontend sends this token to prove identity.

Never store passwords as plain text. Use a library called bcrypt to "hash" passwords (turns them into scrambled strings that can't be reversed). Use jsonwebtoken for creating and verifying JWTs.

What is a Database & Which to Pick?
‚ñº

Right now your server stores data in memory ‚Äî it disappears when you restart. A database persists data permanently. There are two main types: SQL databases (like PostgreSQL, MySQL) store data in neat tables with rows and columns ‚Äî very structured. NoSQL databases (like MongoDB) store data as flexible documents (like JSON).

Recommendation: Start with MongoDB if you want the fastest path (it feels natural with JavaScript since everything is JSON). Learn PostgreSQL next, as it's the most popular database in the real world and is a critical skill.

Connect Database to Your Express Server
‚ñº

Use Mongoose (for MongoDB) or Sequelize/Prisma (for PostgreSQL) to connect your Express server to your database. These libraries let you read, write, update, and delete data using JavaScript ‚Äî no raw database queries needed (though learn SQL eventually too).

Mini Project: Connect MongoDB to your Express server. Create a "Users" collection. Write routes that can: create a user (POST), get all users (GET), get one user by ID, update a user, and delete a user. These are called CRUD operations ‚Äî the foundation of almost every app.
